# 災害・工事情報共有マップ Ver.2

地図上に「道路工事」「土砂崩れ」「浸水」「通行止め」などの情報をピン留めして、職場のメンバー同士でリアルタイムに共有できるWebアプリです。インストール不要、`index.html` をブラウザで開くだけで動きます。

> **Ver.2 の目玉：招待用QRコードで共有**
> 代表者が招待QRを発行すれば、メンバーは**QRを読み込むだけ**で共有に参加できます（各端末での設定入力は不要）。
> さらに**招待PIN**で、QRの発行を代表者だけに限定できます。

---

## 主な機能

| 機能 | 説明 |
|------|------|
| 📍 情報のピン留め | 地図をタップして「種類・タイトル・詳細・期間・登録者」を登録 |
| 🔄 リアルタイム共有 | 誰かが登録・編集・削除すると全員の地図に即時反映（共有モード時） |
| 🏷️ カテゴリフィルタ | 上部チップで種類ごとに表示ON/OFF |
| ➕ カテゴリ追加・削除 | ⚙️設定から独自カテゴリ（例：倒木🌲）を追加／削除可能 |
| 📋 一覧表示 | 登録情報を新しい順にリスト表示、タップで地図移動 |
| 🔍 市町村検索 | 地名で地図を移動（OpenStreetMap Nominatim） |
| 📱 招待QRコード | スキャンするだけで共有に参加。PINで発行を制限可能 |
| 💾 バックアップ | 全データをJSONファイルに書き出し |
| 📲 PWA対応 | ホーム画面に追加してアプリのように使える／オフラインでも起動 |

---

## 2つの動作モード

| モード | バッジ | 保存先 | 用途 |
|--------|--------|--------|------|
| **ローカルモード** | 🟧 ローカル | その端末のみ（localStorage） | 1人で使う・お試し |
| **共有モード** | 🟩 共有中 | Firebase Realtime Database | 複数人でリアルタイム共有 |

初期状態は**ローカルモード**です。共有するには代表者が1回だけFirebaseを設定します（→ [共有設定手順書_Ver2.md](共有設定手順書_Ver2.md)）。

## 使い方（基本操作）

| 操作 | 方法 |
|------|------|
| 情報を登録 | 右下の「＋」→ 地図上の場所をタップ → 入力して保存 |
| 詳細を見る | アイコンをタップ |
| 編集・削除 | アイコンをタップ → ✏️編集 / 🗑削除 |
| 種類で絞り込み | 上部のカテゴリチップをタップでON/OFF |
| 一覧表示 | 右下の📋 |
| 現在地へ移動 | 右下の📍 |
| 市町村検索 | 右上の🔍 |
| カテゴリ追加 | ⚙️ → カテゴリの追加（名前＋絵文字） |
| カテゴリ削除 | ⚙️ → 追加したカテゴリ一覧の「削除」ボタン（標準カテゴリは削除不可） |
| バックアップ | ⚙️ → データをファイルに書き出す |

### 標準カテゴリ

🚧 道路工事 ／ ⛰️ 土砂崩れ ／ 🌊 浸水 ／ ⛔ 通行止め ／ 📌 その他

## 技術スタック

- **地図**：[Leaflet](https://leafletjs.com/) 1.9.4 ＋ OpenStreetMap タイル
- **データ共有**：Firebase Realtime Database（compat SDK 10.12.0）
- **地名検索**：OpenStreetMap Nominatim API
- **QRコード生成**：qrcode.js
- **PWA**：Service Worker によるアプリシェルキャッシュ
- ビルド不要・フレームワーク不使用（ピュアなHTML/CSS/JS）

## セットアップ

### 1人で使う（ローカルモード）
`index.html` をブラウザで開くだけ。設定は不要です。

### 複数人で共有する（共有モード）
代表者が1人だけFirebaseを設定します。詳しい手順は **[共有設定手順書_Ver2.md](共有設定手順書_Ver2.md)** を参照してください（所要約10分・無料）。

招待QR方式を使うには、`index.html` を**Web上に公開**してURLでアクセスする必要があります（無料のNetlify等）。

### スマホでアプリのように使う
- **iPhone**：Safariで開く → 共有ボタン → 「ホーム画面に追加」
- **Android**：Chromeで開く → メニュー → 「ホーム画面に追加」

---

## セキュリティ上の注意

- **招待QR自体が「鍵」です。** 読み取った人は誰でも情報の閲覧・登録・編集ができます。職場外に出さないでください。
- 招待PINは「QR発行ボタンの保護」であり、運用上の抑止です。データベースのルールが誰でも読み書き可（`.read/.write: true`）の場合、技術的に完全な権限制御ではありません。
- **個人情報・機密情報は登録しないでください。**

## Ver.1 との違い

| | Ver.1 | Ver.2 |
|---|---|---|
| 共有への参加 | 各端末で設定コードを手入力 | 招待QRを読むだけ |
| 発行の制限 | なし | 招待PINで代表者に限定可 |
| PWA（ホーム追加・オフライン） | ― | 対応 |
| カテゴリ削除 | ― | 対応 |

手入力の旧方式が必要な場合は Ver.1（`../Ver.１/map_Ver1.html`）をご利用ください。
