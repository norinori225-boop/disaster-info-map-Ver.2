# 災害・工事情報共有マップ【Ver.2】共有設定手順書

> **Ver.2 の特徴：招待用QRコードで共有**
> 代表者がキーを設定して招待QRを発行すれば、メンバーは**QRを読み込むだけ**で共有に参加できます（各端末での手入力は不要）。
> さらに、**招待PIN**を設定すると、QRの発行を代表者だけに限定できます。
>
> ※ 各端末で手動入力する旧方式が必要な場合は **Ver.1** をご利用ください。

---

## 概要

`map_Ver2.html` をブラウザで開くだけで地図アプリが使えます。
初期状態は**ローカルモード**（その端末だけに保存）です。
職場の複数人で共有するには、以下の手順でFirebase（Googleの無料クラウド）を設定します。
**設定作業は代表者1人が1回だけ**行えばOKです。所要時間は約10分です。

---

## 手順1：Firebaseプロジェクトを作成（代表者のみ）

1. https://console.firebase.google.com を開き、Googleアカウントでログイン
2. 「プロジェクトを作成」をクリック
3. プロジェクト名を入力（例：`saigai-map`）→ 続行
4. Googleアナリティクスは「無効」でOK → 「プロジェクトを作成」

## 手順2：Realtime Database を作成

1. 左メニュー「構築」→「Realtime Database」→「データベースを作成」
2. ロケーションは「シンガポール」等でOK
3. セキュリティルールは「**テストモードで開始**」を選択 → 有効にする
4. 作成後、「ルール」タブを開き、以下に書き換えて「公開」を押す
   （テストモードは30日で期限切れになるため必須です）

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

> ⚠️ この設定はURLを知っている人なら誰でも読み書きできる状態です。
> 職場内の限定共有なら実用上問題ありませんが、個人情報や機密情報は登録しないでください。

## 手順3：設定コードを取得

1. Firebase画面左上の歯車 →「プロジェクトの設定」
2. 下の方の「マイアプリ」で **「</>」（ウェブ）** アイコンをクリック
3. アプリ名を入力（例：map）→「アプリを登録」
4. 表示されるコードの中の `const firebaseConfig = { ... }` の **{ } の中身** をコピー

## 手順4：アプリに貼り付け（代表者のみ）

1. `map_Ver2.html` を開き、右上の **⚙️** をタップ
2. Firebase画面に表示されたコードを**そのまま丸ごとコピーして貼り付け**でOKです
   （`const firebaseConfig = { ... }` を含んだままで構いません。アプリが自動で読み取ります）

> ※ `databaseURL` が含まれていることだけ確認してください。
> 含まれていない場合は、Realtime Database作成後にもう一度
> 「プロジェクトの設定」→「マイアプリ」のコードを表示し直してください。

3. 「設定を保存して再読み込み」を押す
4. 右上のバッジが「**共有中**」（緑）になれば成功

---

## 手順5：招待PINを設定（任意・推奨）

招待QRを「代表者だけが発行できる」ようにするための設定です。

1. ⚙️ 設定画面の「**招待用PIN**」欄に、好きな番号や文字（4文字以上）を入力
2. 「**PIN設定**」を押す → ステータスが「🔒 設定済み」になれば完了
3. 以後、誰かが招待QRを表示しようとすると **PIN入力を求められます**
   - 正しいPINを知っている代表者だけがQRを発行できます
   - PINはQRやデータには含まれないため、メンバーには伝わりません

> **PINを解除したいとき**：PIN欄を空欄のまま「PIN設定」を押す → 確認ダイアログでOK
> （解除すると、共有に参加した人は誰でもQRを発行できる状態に戻ります）

---

## 手順6：招待QRを発行してメンバーに配布

> ⚠️ 招待QRは **Web公開したURL**（後述）で開いてから作成してください。
> パソコンでファイルを直接開いた状態（`file:`）では発行できません。

1. ⚙️ →「**📱 招待用QRコードを表示**」をタップ
2. （PIN設定済みの場合）PINを入力
3. 表示されたQRコードを、各メンバーにスマホで読み取ってもらう
4. メンバーがQRを読むと**設定済みの地図が自動で開き**、そのまま共有に参加できます
   - メンバー側での設定コード入力は**不要**です
5. 以後、誰かが登録すると**全員の地図に即時反映**されます

> **このQR自体が「鍵」です。** 読み取った人は誰でも情報の閲覧・登録・編集ができます。
> 職場外に出さないよう注意してください。

---

## 使い方

| 操作 | 方法 |
|---|---|
| 情報を登録 | 右下の「＋」→ 地図上の場所をタップ → 入力して保存 |
| 詳細を見る | アイコンをタップ |
| 編集・削除 | アイコンをタップ → ✏️編集 / 🗑削除 |
| 種類で絞り込み | 上部のカテゴリチップをタップでON/OFF |
| 一覧表示 | 右下の📋 |
| 現在地へ移動 | 右下の📍 |
| カテゴリ追加 | ⚙️ → カテゴリの追加（例：倒木 🌲） |
| バックアップ | ⚙️ → データをファイルに書き出す |

## 費用

Firebaseの無料枠（Sparkプラン）は同時接続100人・保存1GBまで無料。
この用途なら課金が発生することはまずありません。クレジットカード登録も不要です。

## スマホでアプリのように使う

- **iPhone**: Safariでページを開く → 共有ボタン → 「ホーム画面に追加」
- **Android**: Chromeで開く → メニュー → 「ホーム画面に追加」

※ 招待QR方式を使うには、`map_Ver2.html` を**Web上に公開**してURLでアクセスする必要があります（無料のNetlify等）。公開方法が分からない場合はご相談ください。

---

## セキュリティに関する注意（Ver.2）

- 招待PINは「QR発行ボタンの保護」です。**一度QRを受け取った人**は閲覧・登録・編集が可能になります。
- データベースのルールが `".read": true / ".write": true`（誰でも読み書き可）のため、PINは**運用上の抑止**であり、技術的に完全な権限制御ではありません。
  - より厳格に制御したい場合は、Firebase認証を使ったルール設定が必要です（要相談）。
- 個人情報・機密情報は登録しないでください。
